# 确定性边界

人或者 Agent 的行为有或多或少的随机性。
确定性管理的意义在于将方向有差别、行为不确定的人或 Agent 聚集在一起，实现某种确定的目标。

## 并非 100% 确定

只要 出错概率 * 出错后果 小于特定阈值，就可以在实务中认为是确定的。
两条路径：降低出错概率、减小出错后果。

## 心智模型

人在持续预测着外部世界的变化。
如果预测准确，人能以更加自信、高效的方式行动。
如果预测不准确，人会感到不安、紧张、沮丧，并调整预测方式和实际行动加以应对。

Harness 的意义不仅在于增进模型表现与避免常见错误，更在于为 Agent 的人类用户提供一个更为确定的心智模型。
